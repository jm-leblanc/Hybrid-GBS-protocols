## Checking my existing VCF file to see if it's appropriate for GBS to ploidy

GBS to Ploidy needs:
* A filtered VCF file that was mapped to an aligned genome
* Estimates of heterozygosity at each variant
* Allele depth information

# Check the header for FORMAT fields
grep "##FORMAT" fescue_final_maf05_indiv70_var80.recode.vcf

# Look at a few actual data lines to see what's available
grep -v "^#" fescue_final_maf05_indiv70_var80.recode.vcf | head -5

This creates readouts like `1/1:27,3,0`. The 1/1 (or 1/0 or 0/0) shows the homo- or heterozygosity at that variant. The other numbers are a measure of the confidence we have that the zygosity is correct

This shows: 
* the **Chromosome** (`NC_067244.2`)
* the **position on that chromosome** (`19111`)
* the **Reference Allele** (i.e., what the allele is on the reference genome, in this case, `C`)
* the **Variant Allele** (i.e. the variant found in my sample, in this case, `A`
* and then the depth reads for those alleles for each sample
  * e.g., `(0, 0)` in the first position means that Sample 1 has 0 reads for C and 0 reads for A at that position. but something like `(0,3)` means that the individual has 3 reads for A at that position

So it looks like we have the information necessary to run gbs2ploidy.

## Running initial steps in an interactive R session

This could be intensive, so it's probably better off run as a batch script, but there are some things I want to check before proceeding with a batch job

One thing that is required for gbs2ploidy is for your input file to be in **Heterozygous Allele Depth (HAD)** format. However, gbs2ploidy does not include this function, so we use **vcf2ploidy** to convert our file:

```bash
https://github.com/dandewaters/vcf2ploidy
```

# Install required packages

Once inside R, install the required packages:

```bash
install.packages("rjags", repos = "https://cloud.r-project.org")
install.packages("gbs2ploidy", repos = "https://cloud.r-project.org")
```

**IMPORTANT:** Make sure you specify the repository with `repos = "https://cloud.r-project.org"`. Otherwise, R will cycle through every available repository trying to install the packages, and never actually install them

For **vcf2ploidy** you need access to the specific GitHub repository:

```bash
install.packages("remotes", repos = "https://cloud.r-project.org")
remotes::install_github("dandewaters/vcf2ploidy")
```
**NOTE:** The above package has a bunch of dependencies, and those dependencies have their own dependencies that will also be installed automatically. This takes a lot of time. You may want to run this as a batch job rather than an interactive job. Apparently, once you install it once, you won't have to install it again if they're stored in a personal library.
