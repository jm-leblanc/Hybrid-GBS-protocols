### Checking the file to make sure it can be converted to MPGL

```bash
[lebla179@cedar2 entropy]$ # See all FORMAT field definitions
grep "##FORMAT=" fescue_biallelic_snps_only.recode.vcf
##FORMAT=<ID=PL,Number=G,Type=Integer,Description="List of Phred-scaled genotype likelihoods">
##FORMAT=<ID=DP,Number=1,Type=Integer,Description="Number of high-quality bases">
##FORMAT=<ID=SP,Number=1,Type=Integer,Description="Phred-scaled strand bias P-value">
##FORMAT=<ID=AD,Number=R,Type=Integer,Description="Allelic depths (high-quality bases)">
##FORMAT=<ID=ADF,Number=R,Type=Integer,Description="Allelic depths on the forward strand (high-quality bases)">
##FORMAT=<ID=ADR,Number=R,Type=Integer,Description="Allelic depths on the reverse strand (high-quality bases)">
##FORMAT=<ID=GT,Number=1,Type=String,Description="Genotype">
```

The `PL` field is what we're looking for here; this is what the conversion script will extract to make the VCF into an MPGL

## Running the r script

my script to run the conversion is `VCFtoMPGL.R`. It is modified from `inputdataformat_Entropy.R` found at this repository: https://github.com/sonia23cld/Entropy?tab=readme-ov-file which is modified from V. Shastry's script. You can run it in an interactive R environment, it doesn't take too long. 

```bash 
Rscript VCFtoMPGL.R
```


This outputs several files:

* cinds_fescue_biallelic_snps_only.recode.txt  
* fescue_biallelic_snps_only.recode.mpgl  # The MPGL file entropy needs
* ploidy_inds.txt  # The index of ploidy for each sample. We are treating them all as diploid, so each entry is simply "2"
* pntest_mean_gl.txt  
* qk2inds.txt  
* qk3inds.txt  
* qk4inds.txt

From the GitHub, some code to check the output worked correctly:

`wc -l fescue_biallelic_snps_only.recode.mpgl` This number should be one greater than the number of loci in your data set

```bash
23463 fescue_biallelic_snps_only.recode.mpgl
```

One more than my number of loci (23462), so far so good

'awk -F' ' 'NR==3{print NF}' fescue_biallelic_snps_only.recode.mpgl' This number should be one greater than the number of genotype classes times the number of individuals in your data set. This should be true for all lines in your input data file after line number 3.
formula: (ploidy + 1) × number_of_individuals + 1

my output: 292

because I'm treating all my individuals as diploids, this number is correct
Diploid: (2+1) × 97 + 1 = 292
